数字签名

签名的生成和验证

  1. 签名

    • 有原始数据对其进行哈希运算 -> 散列值
    • 使用非对称加密的私钥对散列值加密 -> 签名
    • 将原始数据和签名一并发送给对方

  2. 验证

    • 接收数据

      • 原始数据
      • 数字签名

    • 数字签名, 需要使用公钥解密, 得到散列值

    • 对原始数据进行哈希运算得到新的散列值

小结：

  1. 数据通信
    • 公钥加密, 私钥解密
  2. 数字签名:
    • 私钥加密, 公钥解密

使用椭圆曲线进行数字签名

椭圆曲线在go中对应的包: import “crypto/elliptic”

使用椭圆曲线在go中进行数字签名: import “crypto/ecdsa”

美国FIPS186-2标准, 推荐使用5个素域上的椭圆曲线, 这5个素数模分别是:

P192 = 2^192^ - 2^64^ - 1

P224 = 2^224^ - 2^96^ + 1

P256 = 2^256^ - 2^224^ + 2^192^ - 2^96^ -1

P384 = 2^384^ - 2^128^ - 2^96^ + 2^32^ -1

P512 = 2^512^ - 1
